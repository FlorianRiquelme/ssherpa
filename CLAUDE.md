# ssherpa

## Changelog Policy (Mandatory)

**Claude MUST update CHANGELOG.md under `## [Unreleased]` when making changes.**

Add entries to the appropriate category:
- **Added** - New features
- **Changed** - Changes to existing functionality
- **Deprecated** - Features that will be removed
- **Removed** - Removed features
- **Fixed** - Bug fixes
- **Security** - Security fixes

Write entries from a user perspective (what changed, not how).
